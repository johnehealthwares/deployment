### Issues Encountered & Resolutions

| Issue | Resolution |
|---|---|
| `declare -A` unsupported on macOS bash 3.2 | Replaced with `case` statement or `get_field()` function |
| Private GitHub repos require auth on build instance | Switched from `git clone` to rsync, then back to git clone (repos are public) |
| Wrong local repo paths (`../identity` vs `../rxsoft-identity`) | Fixed SOURCE_MAP with correct paths |
| `rsync` connection drops on large source dirs | Switched to `git clone` on build server with `deployment/` repo |
| Build context resolve error (`deployment/ehealthwares` not found) | Cloned source repos inside `deployment/` directory |
| Missing `nest-cli.json` for healthcare-interop | Added fix step in build script |
| Docker Hub 500 transient error | Added retry loop (3 attempts) |
| rxsoft-backend OOM with `--max-old-space-size=1536` | Increased to 4096 for t3.large |
| Missing `nginx-default.conf` in admin build context | Added copy step in build script |
| ECR image name mismatch (`ehealthwares` vs `rxsoft-ehealthwares`) | Added `ecr_repo_for_service()` mapping function |
| Run instance Docker not authenticated to ECR | Manual `aws ecr get-login-password \| docker login` |
| `docker compose` uses old `docker-compose.yml` (not `.prod.yml`) | Explicit `-f docker-compose.prod.yml` flag |
| MongoDB keyfile permission reset by container recreation | Regenerated keyfile, chown 999:999 |
